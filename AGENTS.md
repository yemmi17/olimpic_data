# AGENTS.md

## Назначение проекта

Это учебный статический дашборд по датасету Олимпийских игр. Проект должен оставаться простым, переносимым и пригодным для короткосрочного запуска на тестовом стенде в Docker.

Основной production base path:

```text
/olimpic/
```

Не ломай этот префикс без явного запроса.

## Коммуникация

- Отвечай на русском.
- Пиши кратко, технически и по делу.
- Если изменение затрагивает Docker, nginx, безопасность, структуру данных, публичные URL или deployment-поведение, сначала опиши план и риски.
- После завершения всегда указывай:
  - что изменено;
  - какие файлы затронуты;
  - как проверено;
  - production/security последствия.

## Структура проекта

```text
data/                   # исходные данные, должны быть доступны на сервере
outputs/                # фактические output-артефакты, должны быть доступны на сервере
site/                   # статический frontend
deploy/nginx/           # nginx-конфигурация
olympics_analysis.py    # генерация Excel и site/data.js
Dockerfile
docker-compose.yml
```

## Важные правила

- Не удаляй `data/athlete_events.csv` без явного запроса.
- Не удаляй `outputs/olympics_report.xlsx` без явного запроса.
- Не меняй production route `/olimpic/` без явного запроса.
- Не добавляй внешние CDN или runtime API без объяснения и согласования.
- Не добавляй секреты, токены, пароли, kubeconfig, `.env` в git.
- Не коммить временные файлы, логи, кеши, виртуальные окружения.
- Не превращай проект в сложное приложение без необходимости: это должен быть статический сайт.

## Аналитика

Источник данных:

```text
data/athlete_events.csv
```

Команда пересборки:

```bash
python olympics_analysis.py
```

Ожидаемые outputs:

```text
outputs/olympics_report.xlsx
site/data.js
```

При изменении логики анализа проверяй:

- ключевые ответы в Summary;
- генерацию Excel;
- генерацию `site/data.js`;
- что сайт открывается и графики не пустые.

## Frontend

- Сайт должен работать без внешнего интернета.
- Все assets должны быть локальными.
- До клика `Показать дашборд` верхние ссылки скрыты.
- После клика появляется дашборд и ссылки `Excel-отчет`, `Код анализа`.
- Развернутый просмотр карточки не должен дублировать заголовок.
- Графики не должны иметь горизонтальный overflow на laptop/mobile viewport.

## Docker / deployment

Контейнер должен отдавать сайт под:

```text
http://<host>:<port>/olimpic/
```

Внутренний порт контейнера:

```text
8080
```

Проверки:

```bash
docker compose config
docker compose up -d --build
curl -I http://127.0.0.1:${OLIMPIC_HOST_PORT:-8088}/olimpic/
```

Если Docker недоступен, укажи это явно и напиши команды, которые нужно выполнить на стенде.

## Security baseline

Перед публикацией проверяй:

```bash
rg -n --hidden --glob '!.git/**' "(password|passwd|secret|token|api[_-]?key|private|BEGIN .*KEY|ghp_|github_pat|AKIA|credential|kubeconfig|\\.env)" .
```

Ожидаемо: секретов нет.

Nginx должен выставлять security headers. Не удаляй их без явного запроса.
