# Дашборд рынка труда: простое описание

## Что это за проект

Это интерактивный дашборд для анализа вакансий по данным портала «Работа России».

Он помогает быстро ответить на вопросы:

- какие специалисты чаще всего нужны работодателям;
- сколько вакансий подходит людям без опыта;
- в каких отраслях больше всего спрос на начинающих сотрудников;
- как отличаются зарплаты по профессиям и уровню опыта;
- какие вакансии есть во Владимирской области;
- по каким профессиям может быть риск оттока кадров в другие регионы;
- есть ли дисбаланс между тем, кого выпускают учебные заведения, и тем, кто нужен работодателям.

Проект сделан как веб-страница: пользователь открывает ссылку в браузере и работает с фильтрами, графиками и таблицами.

## Из чего состоит решение

Решение состоит из трех основных частей.

1. Файл с данными

   Это CSV-файл с вакансиями:

   ```text
   data/vacancy.csv
   ```

   В нем лежат строки с вакансиями: регион, название вакансии, отрасль, зарплата, опыт, дата публикации и другие поля.

2. Программа обработки данных

   Файл:

   ```text
   labor_market_dashboard/data_loader.py
   ```

   Он читает большой CSV-файл, берет только нужные колонки и приводит данные к удобному виду:

   - зарплаты превращает в числа;
   - опыт приводит к понятным категориям: `Без опыта`, `1-3 года`, `3-6 лет`, `6+ лет`;
   - даты публикации превращает в нормальный формат дат;
   - названия колонок из исходного файла приводит к единой структуре.

3. Веб-дашборд

   Файл:

   ```text
   labor_market_dashboard/app.py
   ```

   Он показывает интерфейс: фильтры, показатели, графики и сводные таблицы.

## Как работает дашборд

Когда пользователь открывает страницу, приложение делает несколько шагов:

1. Проверяет, есть ли полный файл:

   ```text
   data/vacancy.csv
   ```

2. Если файл есть, приложение читает его.

3. Если полного файла нет, приложение использует небольшой демонстрационный файл:

   ```text
   labor_market_dashboard/sample_vacancies.csv
   ```

4. Данные очищаются и приводятся к единому виду.

5. Пользователь выбирает фильтры: регион, отрасль, опыт, зарплату, дату и профессию.

6. Дашборд перестраивает графики и таблицы под выбранные фильтры.

## Что показывает дашборд

### Основные показатели

В верхней части показываются короткие числовые показатели:

- общее количество вакансий в выбранном срезе;
- количество вакансий без опыта;
- доля вакансий без опыта;
- медианная зарплата.

Медианная зарплата означает «типичное значение»: половина вакансий выше этой суммы, половина ниже.

### График вакансий без опыта по отраслям

Показывает, в каких отраслях чаще всего ищут сотрудников без опыта.

Например, это помогает понять, где проще начать карьеру после учебы или переобучения.

### Диаграмма по опыту

Показывает, какая доля вакансий требует опыт, а какая доступна новичкам.

Это полезно для оценки доступности рынка труда для выпускников и начинающих специалистов.

### Динамика публикаций

Показывает, как менялось количество опубликованных вакансий по месяцам.

По этому графику можно увидеть рост, спад или сезонность спроса.

### Зарплаты по опыту

Показывает разброс зарплат для разных уровней опыта.

Так можно увидеть, насколько зарплата зависит от стажа и насколько рынок готов платить новичкам.

### Сводная таблица

Сводная таблица показывает количество вакансий по отраслям и опыту.

Она нужна для более точного анализа, когда графика недостаточно и нужно посмотреть конкретные числа.

### Блок про возможный отток кадров

Этот блок сравнивает зарплаты по профессиям во Владимирской области и в других регионах.

Если по какой-то профессии в других регионах зарплата заметно выше, это может быть сигналом риска оттока кадров.

Важно: это не доказывает сам факт оттока. Это аналитическая гипотеза, которую можно использовать для дальнейшего исследования.

### Блок про выпускников вузов

В этот блок можно загрузить отдельный CSV-файл с количеством выпускников по специальностям.

Формат:

```csv
specialty,graduates
Junior analyst,120
Process engineer,80
Teacher,160
```

После загрузки дашборд сравнит:

- сколько выпускников есть по специальности;
- сколько вакансий есть по похожему направлению;
- где спрос выше выпуска;
- где выпуск может превышать спрос.

Этот блок нужен для анализа дисбаланса между образованием и рынком труда.

## Что важно объяснить заказчику

Главная идея проекта простая:

> Мы берем открытые данные о вакансиях, очищаем их, приводим к понятному виду и показываем в интерактивном дашборде, чтобы можно было быстро анализировать рынок труда региона.

Дашборд не заменяет полноценное социологическое исследование, но хорошо подходит как аналитический инструмент:

- для первичного анализа рынка труда;
- для выявления проблемных отраслей;
- для оценки спроса на молодых специалистов;
- для поиска профессий с риском кадрового дефицита;
- для подготовки презентации или отчета.

## Где должен лежать файл данных

На сервере файл с вакансиями нужно положить сюда:

```text
data/vacancy.csv
```

То есть рядом с проектом должна быть такая структура:

```text
PANDAS/
  docker-compose.yml
  data/
    vacancy.csv
```

Файл не хранится в git, потому что он большой и может регулярно обновляться.

## Как обновить данные

Чтобы обновить данные:

1. Скачать новый CSV-файл с вакансиями.
2. Заменить старый файл:

   ```text
   data/vacancy.csv
   ```

3. Перезапустить сервис:

   ```bash
   docker compose restart labor-market-dashboard
   ```

Если менялся только CSV-файл, пересобирать Docker-образ обычно не нужно.

## Как запустить на сервере

Из корня проекта:

```bash
docker compose up -d --build labor-market-dashboard
```

После запуска дашборд будет доступен по адресу:

```text
http://<server-ip>:8501/
```

Если сервер закрыт firewall или стоит reverse proxy, внешний адрес может быть другим.

## Как проверить, что все работает

Проверить состояние сервиса:

```bash
docker compose ps
```

Проверить healthcheck:

```bash
curl http://127.0.0.1:8501/_stcore/health
```

Ожидаемый ответ:

```text
ok
```

Посмотреть логи:

```bash
docker compose logs --tail=100 labor-market-dashboard
```

## Ограничения

- Качество анализа зависит от качества исходного CSV-файла.
- Если в исходных данных есть ошибки, они могут повлиять на графики.
- Блок про отток кадров показывает гипотезу на основе зарплат и вакансий, а не фактическую миграцию людей.
- Для точного анализа вузов нужны отдельные реальные данные о выпускниках.
- Дашборд показывает картину по загруженному файлу, а не подключается к порталу «Работа России» в реальном времени.
